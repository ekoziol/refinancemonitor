# Code Review Guidelines

Review this merge request focusing on:

## Code Quality
- Clean, readable code following project conventions
- Appropriate error handling
- No dead code or debugging artifacts

## Security
- Input validation at boundaries
- No hardcoded secrets or credentials
- Safe handling of user data

## Performance
- Efficient algorithms and data structures
- No obvious N+1 queries or performance traps
- Resource cleanup (connections, file handles)

## Testing
- Adequate test coverage for changes
- Edge cases considered
- Tests are clear and maintainable

## Documentation
- Public APIs documented
- Complex logic explained
- README updated if needed

Be specific and actionable. Reference line numbers when possible.
