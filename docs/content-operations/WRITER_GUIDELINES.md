# Writer Guidelines

## Brand Voice & Tone

### Our Voice
RefiAlert is the **trusted friend who happens to be a mortgage expert**. We're knowledgeable but never condescending, helpful but never pushy.

### Voice Characteristics

| Do | Don't |
|----|-------|
| Be confident and authoritative | Be arrogant or dismissive |
| Use clear, simple language | Use unnecessary jargon |
| Be encouraging and supportive | Be condescending or preachy |
| Acknowledge complexity when it exists | Oversimplify important details |
| Use "you" to address the reader | Use passive voice excessively |

### Tone by Content Type

**Educational Content**
- Warm, patient teacher
- "Let's walk through this together"
- Heavy use of examples

**Tool/Calculator Pages**
- Practical, efficient
- "Here's exactly what you need"
- Action-oriented

**News/Updates**
- Informative, timely
- "Here's what this means for you"
- Clear implications

**Comparison Guides**
- Objective, balanced
- "Here are the facts"
- Let readers decide

## Content Structure Requirements

### Every Post Must Include

1. **Headline (H1)**
   - 60 characters max
   - Include primary keyword
   - Clear value proposition
   - Avoid clickbait

2. **Introduction (150-200 words)**
   - Hook in first sentence
   - State what reader will learn
   - Establish relevance ("If you're...")
   - Include primary keyword naturally

3. **Body (1500-2500 words)**
   - H2 headings every 200-300 words
   - H3 for subsections
   - Short paragraphs (3-4 sentences max)
   - Bulleted lists for scanability
   - Data and statistics with sources
   - Examples and scenarios

4. **Conclusion (100-150 words)**
   - Summarize key takeaways
   - Include clear CTA
   - Natural next step for reader

5. **Required Elements**
   - 3-5 internal links
   - 2-3 external authority links
   - Calculator/tool integration where relevant
   - At least one image/visual

### Heading Hierarchy

```
# H1: Main Title (only one per post)
## H2: Major Sections
### H3: Subsections
#### H4: Rarely used, only for deep nesting
```

**H2 Examples:**
- "What Is Refinancing?" (not "Refinancing Definition")
- "5 Signs You Should Refinance Now" (not "Signs")
- "How to Calculate Your Break-Even Point" (not "Break-Even Calculation")

## SEO Requirements

### Keyword Usage

**Primary Keyword**
- In H1 headline
- In first 100 words
- In at least one H2
- In meta description
- Natural density (1-2%)

**Secondary Keywords**
- Sprinkle throughout body
- Use in H2/H3 where natural
- Include variations and synonyms

### Meta Tags

**Title Tag (60 chars)**
```
[Primary Keyword]: [Value Proposition] | RefiAlert
```
Example: "Refinance Break-Even Calculator: Find Your Magic Number | RefiAlert"

**Meta Description (155 chars)**
- Include primary keyword
- Clear value proposition
- Include call-to-action
- Make it compelling to click

Example: "Calculate exactly when refinancing pays off with our free break-even tool. Enter your numbers and get your answer in seconds."

### URL Structure
- Use primary keyword
- Keep short (3-5 words)
- Use hyphens, not underscores
- All lowercase

Good: `/mortgage-refinance-guide/`
Bad: `/the-ultimate-complete-guide-to-mortgage-refinancing-in-2026/`

## Quality Standards

### Accuracy
- All statistics must have sources
- Cite sources inline or with footnotes
- Use data less than 2 years old
- Verify legal/regulatory information
- When in doubt, say "typically" or "generally"

### Readability
- Target Flesch Reading Ease: 60+
- Average sentence length: 15-20 words
- Avoid walls of text
- Use transition words
- Break up with visuals

### Originality
- No plagiarism (run Copyscape)
- Don't copy competitor structure exactly
- Add unique perspective/analysis
- Use original examples when possible

## Formatting Guidelines

### Text Formatting

**Bold** for:
- Key terms on first use
- Important numbers
- Critical warnings

*Italics* for:
- Emphasis
- Publication names
- Terms being defined

### Lists

Use bulleted lists for:
- 3+ items without order
- Features or benefits
- Quick tips

Use numbered lists for:
- Step-by-step instructions
- Rankings
- Sequences

### Tables

Use for:
- Comparisons
- Data presentation
- Quick reference info

Always include headers and keep mobile-friendly (2-4 columns max).

### Callout Boxes

```
> **Pro Tip:** [Insider advice]
```

```
> **Warning:** [Important caution]
```

```
> **Example:** [Real-world scenario]
```

## Content Types

### Educational Guides
- Comprehensive coverage
- 2000-3000 words
- Multiple H2 sections
- FAQ section at end
- Heavy internal linking

### How-To Posts
- Step-by-step format
- 1500-2000 words
- Numbered steps
- Screenshots/visuals
- Success criteria clear

### Comparison Posts
- Objective analysis
- 1500-2500 words
- Comparison table
- Pros/cons for each option
- Clear recommendation at end

### News/Updates
- Timely information
- 800-1200 words
- What it means for reader
- Action items if any
- Update date prominent

### Tool/Calculator Pages
- Feature explanation
- 1000-1500 words
- How to use instructions
- Interpretation of results
- Related resources

## Writing Process

### Before Writing
1. Review content brief thoroughly
2. Research topic (30-60 min)
3. Review competitor content
4. Outline your approach
5. Gather statistics/sources

### During Writing
1. Write introduction last (after body)
2. Focus on one section at a time
3. Don't self-edit while drafting
4. Leave [NOTES] for fact-checking
5. Include [CTA] placeholders

### After Writing
1. Step away for 1+ hour
2. Read aloud for flow
3. Run Grammarly check
4. Check Hemingway score
5. Verify all links work
6. Submit for review

## Common Mistakes to Avoid

### Language
- "In order to" → "To"
- "Due to the fact that" → "Because"
- "At this point in time" → "Now"
- "It is important to note" → [just state it]
- "Very" → [find stronger word]

### Structure
- Starting every paragraph with "I"
- Burying the lead
- Missing transitions between sections
- Inconsistent heading styles
- Orphan sentences

### SEO
- Keyword stuffing
- Exact-match anchor text everywhere
- Ignoring search intent
- Writing for robots, not humans
- Thin content to hit word count
